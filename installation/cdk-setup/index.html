# AWS CDK Setup

# Overview

This document outlines the steps to set up and work with the AWS Cloud Development Kit (CDK) in an existing project. AWS CDK simplifies creating, managing, and deploying AWS resources using familiar programming languages.

# Prerequisites

Before proceeding, ensure you have the following:

  • Node.js (>= 10.3)
  • Python (>= 3.6)
  • AWS Command Line Interface (CLI)
  • An AWS account and an IAM user with the necessary permissions

# Installing AWS CDK

  1. Install Node.js: Download and install Node.js from nodejs.org.

  2. Install AWS CDK: Run the following command to install AWS CDK globally:

    npm install -g aws-cdk

  3. Verify installation: Run the following command to verify that AWS CDK is installed correctly:

    cdk --version

# Configuring AWS Credentials

  1. Create IAM user: Create an IAM user with the necessary permissions to access AWS resources. For more information, see Creating an IAM User in Your AWS Account.

  2. Configure AWS CLI: If not already done, run:

    aws configure

    Enter the access key ID and secret access key of the IAM user created in the previous step. You can also specify the default region and output format.

# Cloning the Repository

  1. Clone the Project: If the CDK project is hosted in a version control system (like Git), clone the repository to your local machine.
git clone git@gitlab.com:rootstack/rootlenses/rootlenses.git

# Initializing the CDK Project

To manually create a virtualenv on MacOS and Linux:

$ python3 -m venv .venv

After the init process completes and the virtualenv is created, you can use the following step to activate your virtualenv.

$ source .venv/bin/activate

If you are a Windows platform, you would activate the virtualenv like this:

% .venv\Scripts\activate.bat

Once the virtualenv is activated, you can install the required dependencies.

$ pip install -r requirements.txt

At this point you can now synthesize the CloudFormation template for this code.

# CDK Bootstrap

The bootstrap process prepares your AWS environment for use with AWS CDK applications. It creates the necessary resources in AWS for deploying your application.

# Executing Bootstrap

To perform the bootstrap, run the following command in your terminal:

cdk bootstrap aws://ACCOUNT_ID/REGION

Replace ACCOUNT_ID with your AWS account ID and REGION with the region you want to work in.

# CDK Deploy

The cdk deploy command is used to deploy your CDK application to AWS. This will create or update resources defined in your stack.

# Deploying a Stack

To deploy a specific stack, use the command:

$ cdk deploy stackName

# Deploying Specific Stacks

In your CDK application, you may have multiple stacks defined. Here's how to deploy each of the specific stacks in your project.

# Deploy_bucket

The Deploy_bucket stack is used to create and configure an S3 bucket.

To deploy this stack, run:

cdk deploy deployBucket

# CreateCatalogDatabase

The CreateCatalogDatabase stack is responsible for creating a catalog database in AWS Glue.

To deploy this stack, run:

cdk deploy createCatalog

# SetETLJob

The CdkAwsProjectStack may contain various configurations for your AWS project.

To deploy this stack, run:

cdk deploy deployScript

# Useful commands

  • cdk ls list all stacks in the app
  • cdk synth emits the synthesized CloudFormation template
  • cdk deploy deploy this stack to your default AWS account/region
  • cdk diff compare deployed stack with current state
  • cdk docs open CDK documentation
